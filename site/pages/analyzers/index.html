Why it matters

Manual transcription of analyzer results is slow, error-prone, and frustrating for staff — and every transcription error is a potential patient-safety event. Automated interfaces eliminate that risk while freeing technologists to focus on interpretation and quality, not data entry.

How it works

Bidirectional communication

Send pending test lists to analyzers and receive results back automatically. Work orders download; results upload — no manual step in between.

Broad protocol support

ASTM E1381/E1394 (LIS2-A2) and HL7 v2.5.1 cover the vast majority of instruments, with CSV/flat-file import for legacy and point-of-care devices.

Visual field mapping

Map analyzer test codes to OpenELIS tests through a graphical interface — no programming. Add new analyzers in hours, not weeks, without a server restart.

Quality control built in

Instrument QC status, Levey-Jennings charts, and Westgard-rule monitoring surface right alongside the analyzer connection.

Pre-configured instruments — and a plugin model for the rest

Common analyzers like GeneXpert, Sysmex, Cobas, and QuantStudio ship with ready-to-use mappings you can deploy immediately. For anything else, OpenELIS uses a modular plugin architecture — each analyzer type has its own hot-deployable plugin, and you can develop custom plugins for any ASTM-, HL7-, or file-based instrument.

Why it matters — "We already own that analyzer" is never a blocker. If it speaks ASTM, HL7, or exports a file, OpenELIS can take its results — and the open plugin model means the community keeps expanding the list.
